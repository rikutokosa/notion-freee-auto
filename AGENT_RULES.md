# AGENT_RULES

このプロジェクト（notion-freee-auto）における作業規範・完了条件・禁止事項です。
他のAIエージェントが作業を引き継ぐ際も、必ずこのルールに従ってください。

## 最優先原則
- **壊さない**
- **勝手に本番データを変更しない**
- **シークレットを漏らさない**
- **既存の安全機構を無効化しない**

## 作業開始前の必須事項
- 作業開始前に必ず `AGENT_RULES.md` と `MEMORY.md` を読むこと。
- 本ルールと個別の指示が矛盾する場合は、勝手に判断せず、必ずユーザーに確認すること。

## 完了の定義
「対応済み」「完了」と報告するには、必ず以下を提示すること。
- 該当 git diff
- `scripts/selfcheck.sh` の実行結果
- 修正内容に対応する grep / 確認コマンドの実出力
- 実装箇所・呼び出し箇所・削除箇所のdiff
- 本番未実施・実機未実施の明示
- シークレット検出結果は必ず `[REDACTED]` でマスクして提示すること

## 禁止事項
- grepを通すためだけの表面的な修正
- `py_compile` だけで「テスト済み」と報告すること
- 既存の認証、バリデーション、idempotency、dry_run、安全確認を無断で削除・無効化すること
- 本番のfreee / Notion / OpenAIに、ユーザー承認なしで書き込み・削除・送信テストを行うこと
- secret検出結果にトークン値を全文表示すること

## requirementsの扱い
- 本番環境で必要な依存パッケージは `requirements.txt` に記載する。
- 開発・検証用のパッケージ（pyflakes等）は `requirements-dev.txt` に記載する。

## 報告フォーマット
作業完了時の報告には、以下を含めること。
- 変更ファイル一覧
- 要件ごとのdiff
- 検証コマンド
- 検証結果
- 未実施項目
- 本番反映可否
